# Insect Crop Harm Camera Detector — Flutter version

Cross-platform Flutter port of the Android app in the parent folder.
Same idea: live camera → TensorFlow Lite insect classifier → check the detected
insect against the selected crop → show **DANGER** (red + vibration + alert
sound) or **SAFE** (green).

Runs on Android and iOS from a single Dart codebase.

## Prerequisites

1. Install the **Flutter SDK** (stable channel ≥ 3.19):
   https://docs.flutter.dev/get-started/install
2. Run `flutter doctor` and resolve any reported issues.
3. For Android: Android Studio (or just the command-line tools + a JDK 17).
4. For iOS: Xcode + CocoaPods (macOS only).

## First-time setup (one command)

This project ships only `lib/`, `assets/`, `pubspec.yaml` and config. Platform
folders (`android/`, `ios/`, `windows/`, ...) are generated by Flutter so they
always match your installed Flutter version.

From inside `flutter_app/`:

```bash
flutter create .
flutter pub get
```

`flutter create .` fills in the missing platform folders **without touching**
your `lib/`, `assets/`, or `pubspec.yaml`.

## Add the ML model

The repo ships a placeholder at `assets/insect_model.tflite`. Replace it with
a real trained model. See `assets/README_MODEL.txt` for a 5-minute training
walkthrough using Teachable Machine. Specs:

- Input: 224 × 224 × 3, float32, values in `[0, 1]`
- Output: float32 vector of length N (matches lines in `assets/labels.txt`)

If you skip this step the app still launches and shows a `MODEL NEEDED` banner
instead of crashing.

## Platform permissions

After running `flutter create .` you must add camera permissions.

### Android — `android/app/src/main/AndroidManifest.xml`

Add these **inside `<manifest>` and above `<application>`**:

```xml
<uses-permission android:name="android.permission.CAMERA" />
<uses-permission android:name="android.permission.VIBRATE" />
<uses-feature
    android:name="android.hardware.camera"
    android:required="true" />
```

In `android/app/build.gradle` make sure `minSdkVersion` is at least **24** (the
`camera` and `tflite_flutter` plugins both require it):

```groovy
defaultConfig {
    minSdkVersion 24
}
```

### iOS — `ios/Runner/Info.plist`

Add inside the top-level `<dict>`:

```xml
<key>NSCameraUsageDescription</key>
<string>This app uses the camera to identify insects on crops.</string>
```

Set the minimum iOS deployment target to **12.0** in
`ios/Podfile` (uncomment and change the first line):

```ruby
platform :ios, '12.0'
```

## Run on a real device

```bash
# Plug in your phone with USB debugging enabled
flutter devices          # confirm device shows up
flutter run              # build + install + launch
```

On the device: grant camera permission → pick the crop from the dropdown →
point the camera at an insect.

```
├── analysis_options.yaml        lint config
├── assets/
│   ├── labels.txt               insect class names (one per line)
│   ├── insect_model.tflite      placeholder — replace with your model

└── lib/
    ├── main.dart                app entry point + camera UI
    ├── insect_classifier.dart   TFLite inference + YUV/BGRA conversion
    └── crop_risk_database.dart  insect ↔ crop harm lookup
```

## Notes

- The classifier runs at most once every 1.5 seconds to keep the UI smooth.
- Alerts are throttled to once every 5 seconds.
- Confidence below 50% is reported as `LOW CONFIDENCE` (same threshold as the
  Java version).
- The pure-Dart YUV420→RGB conversion in `insect_classifier.dart` is fine for
  `ResolutionPreset.medium` (~480p). If you bump the resolution, consider
  moving that conversion to an isolate to keep the camera stream responsive.
